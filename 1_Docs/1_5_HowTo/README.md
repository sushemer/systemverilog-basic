# 1.5 Howto – Quick guides so you don’t get lost

The `1_5_howto` folder brings together **practical guides** that help you:

- Connect hardware  
- Understand how code flows  
- Run simulation, synthesis, and programming  
- Solve common errors  

---

## 0. Folder contents

- `1_5_1_connections/`  
  Physical connections guide.

- `1_5_2_code_flow/`  
  Explains the flow between:
  - physical top  
  - `hackathon_top`  
  - peripherals  

- `1_5_3_how_to_run/`  
  Explains which scripts to run and how.

- `1_5_4_troubleshooting_and_pitfalls/`  
  Common errors and checklist.

---

## 1. How to use these guides

- Before wiring → `1_5_1_connections`  
- Before programming → `1_5_2_code_flow`  
- Before running scripts → `1_5_3_how_to_run`  
- When something fails → `1_5_4_troubleshooting_and_pitfalls`  

---

## 2. What you don’t need to read every time

- Only hardware → connections  
- Only code → code_flow  
- Only running → how_to_run  
- Only errors → troubleshooting  

---

## 3. Relation with other folders

`1_5_howto` is a bridge between:

- documentation  
- activities, labs, and implementations  

It helps:

- avoid mistakes  
- follow a consistent workflow
